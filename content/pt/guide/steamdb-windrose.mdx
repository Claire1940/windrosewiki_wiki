## Passo 1: Instalando o SteamCMD

O SteamCMD é a versão de linha de comando do cliente Steam. É a ferramenta padrão da indústria para baixar e atualizar arquivos de servidores dedicados. Siga estes passos para preparar seu ambiente:

1.  **Baixe o SteamCMD:** Visite o site oficial da Valve Developer Community e baixe a versão para Windows do SteamCMD.
2.  **Extração:** Crie uma pasta chamada `SteamCMD` na raiz da sua unidade (ex: `C:\SteamCMD`) e extraia o conteúdo do arquivo zip lá.
3.  **Inicialização:** Execute o `steamcmd.exe`. O programa baixará as atualizações necessárias e o levará a um prompt `Steam>`.

## Passo 2: Baixando os Arquivos do Servidor Windrose

Assim que o SteamCMD for inicializado, você precisará puxar os arquivos associados ao ID do servidor **steamdb windrose**. Use a seguinte sequência de comandos no terminal:

1.  Digite `login anonymous` e pressione Enter.
2.  Defina o diretório de instalação digitando `force_install_dir ./windrose_server/`.
3.  Execute o download digitando `app_update 4129620 validate`.

O sinalizador "validate" é importante, pois verifica se há arquivos corrompidos durante o processo de download. Quando o terminal exibir "Success! App '4129620' fully installed", você pode digitar `exit` para fechar o prompt.

## Passo 3: Lançamento Inicial do Servidor

Navegue até a sua pasta `windrose_server`. Você encontrará um executável chamado `WindroseServer.exe` e um arquivo em lote normalmente chamado `StartServerForeground.bat`. 

**Siga estes passos para a primeira execução:**
1.  Clique com o botão direito em `StartServerForeground.bat` e selecione **Editar**.
2.  Certifique-se de que o caminho está correto e execute o arquivo em lote.
3.  Aguarde o console gerar os arquivos do mundo. Você verá logs de "Collector Loading" e "Event".
4.  Assim que o mundo for gerado, pressione `Ctrl + C` na janela do console para desligá-lo com segurança. Este passo é necessário para gerar os arquivos JSON de configuração.

## Passo 4: Configurando as Definições do Servidor

Após o primeiro desligamento, navegue até o diretório `Saved/Config` dentro da pasta do seu servidor. Aqui, você encontrará o `ServerDescription.json`. Este arquivo é onde você define a identidade do seu servidor.

| Parâmetro | Descrição | Configuração Recomendada |
| :--- | :--- | :--- |
| **ServerName** | O nome exibido no navegador de servidores. | Nome da Sua Comunidade |
| **MaxPlayerCount** | Total de slots disponíveis. | **8 a 16** |
| **Password** | Deixe em branco para acesso público. | **Alfanumérico Seguro** |
| **IsPasswordProtected** | Ativa/Desativa a verificação de senha. | true / false |
| **InviteCode** | Um código único usado para conexões diretas. | **Gerado automaticamente** |

> **💡 Dica:** Mantenha sempre um backup do seu `InviteCode`. No Windrose, este código costuma ser a maneira mais confiável para amigos encontrarem seu servidor se o navegador global estiver com tráfego intenso.

## Seção 5: Configuração de Rede e Redirecionamento de Portas

Para permitir que jogadores fora da sua rede local entrem, você deve abrir portas específicas no seu roteador e no Firewall do Windows. Esta é a fase onde a maioria dos anfitriões de servidores encontra problemas.

1.  **Firewall do Windows:** Crie uma Nova Regra de Entrada para os protocolos TCP e UDP.
2.  **Configurações do Roteador:** Faça login no gateway do seu roteador e navegue até a seção de Redirecionamento de Portas (Port Forwarding).
3.  **Portas para Redirecionar:** Por padrão, o Windrose normalmente utiliza portas na faixa **7777-7778** e **27015** para consultas Steam. 

Consulte a documentação específica do fabricante do seu roteador para garantir que o IP interno da sua máquina de servidor esteja mapeado corretamente para essas portas.

Como observado em discussões avançadas de configuração do **steamdb windrose**, existe um segundo arquivo de configuração, mais detalhado, que pode ser criado para manipular as mecânicas de jogo. Este arquivo permite ajustar:

Para implementar essas mudanças, você deve criar um novo arquivo JSON baseado nos modelos fornecidos na pasta `README` ou de documentação do servidor. Copie a estrutura JSON de exemplo, modifique os valores ao seu gosto e salve-o na pasta de configuração.

## Mantendo seu Servidor Windrose

Manter seu servidor atualizado é vital para a compatibilidade. Sempre que os desenvolvedores lançarem um patch, seu servidor provavelmente ficará "Incompatível" até que você atualize os arquivos.

**O Processo de Atualização:**
1.  Desligue o console do servidor.
2.  Abra o SteamCMD.
3.  Execute `login anonymous`.
4.  Execute `app_update 4129620`.
5.  Assim que terminar, reinicie o arquivo em lote do seu servidor.

Você pode automatizar isso criando um arquivo "Update.bat" que contenha esses comandos, permitindo uma manutenção com um único clique.

Para mais informações sobre atualizações do jogo e estatísticas de jogadores, verifique a [Página Oficial do Windrose no Steam](https://store.steampowered.com/app/2431620/Windrose/) para os logs de desenvolvedor mais recentes.

## FAQ

**P: Por que meus amigos não conseguem ver meu servidor no navegador?**

R: Isso geralmente se deve a problemas de redirecionamento de portas. Certifique-se de que as portas 7777 e 27015 estão redirecionadas para TCP e UDP. Além disso, verifique se o Firewall do Windows não está bloqueando o `WindroseServer.exe`.

**P: Preciso comprar uma segunda cópia do jogo para hospedar?**

R: Não. Ao usar o App ID do **steamdb windrose** (4129620) e fazer login como "anonymous" no SteamCMD, você pode hospedar o servidor sem adquirir uma licença adicional.

**P: Como eu mudo o mapa do mundo?**

R: A geração do mundo é tratada durante o lançamento inicial. Se você quiser começar um mundo novo, deve excluir a pasta `Saved` no diretório do seu servidor. No próximo lançamento, uma nova semente aleatória será gerada.

**P: Posso rodar isso em um servidor Linux?**

R: Embora os arquivos nativos sejam para Windows, muitos anfitriões executam com sucesso o servidor dedicado do Windrose em distribuições Linux usando Proton ou Wine. No entanto, a configuração é significativamente mais complexa e pode exigir dependências adicionais como `lib32gcc`.
