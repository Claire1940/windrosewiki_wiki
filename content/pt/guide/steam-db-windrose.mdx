Ao hospedar um servidor dedicado para Windrose, você precisa especificamente do **App ID** para o software do servidor, que geralmente é diferente do ID do cliente do jogo.

Ao consultar a página técnica do **steam db windrose**, podemos identificar que o Servidor Dedicado do Windrose usa um ID específico para a busca via SteamCMD. Isso garante que você baixe os binários corretos, otimizados para o desempenho do servidor em vez do cliente completo do jogo, o que economiza espaço significativo em disco e memória.

## Passo a Passo: Instalando via SteamCMD

Para hospedar um servidor sem possuir uma segunda cópia do jogo, você deve usar o SteamCMD. Esta versão de linha de comando do Steam permite logins anônimos para buscar arquivos de servidor.

### 1. Inicialize o SteamCMD
Baixe o executável do SteamCMD no site oficial da Valve Developer Community. Extraia a pasta para um diretório dedicado em sua unidade (ex: `C:\SteamCMD`). Execute o `steamcmd.exe` para permitir que ele se atualize e inicialize sua estrutura de arquivos interna.

### 2. Busque os arquivos do Windrose
Assim que vir o prompt `Steam>`, siga estas sequências de comandos para baixar os arquivos do servidor usando o ID encontrado no **steam db windrose**:

1.  Digite `login anonymous` e aperte Enter.
2.  Digite `app_update 4129620 validate` e aperte Enter.
3.  Aguarde a mensagem "Success! App '4129620' fully installed".
4.  Digite `exit` para fechar o prompt.

## Configurando os Arquivos do Servidor

Após o término do download, navegue até o diretório do seu SteamCMD: `\steamapps\common\Windrose Dedicated Server\`. Esta pasta contém o executável e os arquivos em lote necessários para iniciar o mundo.

### Estrutura de Arquivos Essencial
*   **Start server foreground.bat**: O script de inicialização principal.
*   **Saved/Config/WindowsServer/**: O diretório onde residem suas configurações personalizadas.
*   **server description.json**: O arquivo usado para definir a identidade do seu servidor.

### Editando a Descrição do Servidor
Abra o arquivo `server description.json` com um editor de texto como o Notepad++. É aqui que você define como seu servidor aparece na lista.

| Parâmetro | Valor de Exemplo | Descrição |
|---|---|---|
| **ServerName** | "Tortuga Bay 2026" | O nome exibido na lista de servidores |
| **MaxPlayerCount** | 8 | O máximo de jogadores simultâneos |
| **Password** | "Pirate123" | Deixe vazio para um servidor público |
| **IsPasswordProtected** | true / false | Ativa ou desativa a exigência de senha |
| **InviteCode** | [Gerado automaticamente] | Usado para conexões diretas |

> **💡 Dica:** Sempre execute o servidor uma vez e depois o feche usando `Ctrl+C` antes de editar as configurações. Isso garante que o jogo gere a estrutura de pastas necessária e os arquivos JSON padrão para você modificar.

## Rede e Redirecionamento de Portas (Port Forwarding)

Para que amigos se conectem ao seu servidor de fora da sua rede local, você deve abrir portas específicas no seu roteador e no Firewall do Windows. Caso contrário, o servidor aparecerá como "Offline" ou dará "Timed Out" para usuários externos.

1.  **Acesse seu Roteador**: Digite o IP do seu gateway (geralmente 192.168.1.1) em um navegador.
2.  **Localize o Port Forwarding**: Geralmente fica em Configurações Avançadas ou Jogos.
3.  **Redirecione as Portas**: Você geralmente precisa redirecionar as portas de consulta (query) e comunicação do jogo (verifique o arquivo `README` na pasta do servidor para o intervalo de portas específico de 2026).
4.  **Regras de Firewall**: Crie uma "Regra de Entrada" no Windows Defender Firewall para permitir que o `WindroseServer.exe` se comunique através da sua rede.

> **⚠️ Nota de Segurança:** Abrir portas acarreta riscos inerentes. Nunca compartilhe o endereço IP direto do seu servidor em fóruns públicos; em vez disso, use o sistema de código de convite fornecido na interface do Windrose.

## Entrando no Servidor Dedicado

Assim que a janela do console mostrar que o mundo foi carregado com sucesso, os jogadores podem entrar usando o menu do jogo.

1.  Inicie o **Windrose**.
2.  Selecione **Jogar** (Play) no menu principal.
3.  Escolha **Conectar a um Servidor** (Connect to a Server).
4.  Cole o **Código de Convite** (Invite Code) encontrado no seu arquivo `server description.json`.
5.  Clique em **Conectar**.

Se o servidor não aparecer, verifique se o seu endereço IP não mudou e se a janela `Start server foreground.bat` ainda está ativa.

## FAQ

**P: Onde posso encontrar o App ID mais recente para o servidor Windrose?**

R: Você deve verificar a página do **steam db windrose** na seção "Apps" ou "Depots". Em 2026, o ID do servidor dedicado é 4129620, mas atualizações de grande porte podem, às vezes, introduzir novos ramos experimentais.

**P: Preciso comprar uma segunda cópia do Windrose para hospedar um servidor?**

R: Não. Ao usar o SteamCMD e fazer login como "anônimo", você pode baixar e hospedar os arquivos do servidor dedicado gratuitamente. No entanto, você ainda precisa de uma cópia comprada do jogo em sua conta principal para realmente jogar no servidor.

**P: O console do meu servidor travou em "Collector Loading". Ele parou de funcionar?**

R: Não necessariamente. Dependendo da velocidade da sua CPU e do tamanho do mundo, a geração do layout inicial da ilha pode levar vários minutos. Verifique o seu Gerenciador de Tarefas; se o uso da CPU para o processo estiver ativo, o servidor ainda está trabalhando.

**P: Como atualizo meu servidor Windrose quando um novo patch é lançado?**

R: Execute o SteamCMD novamente, faça o login anônimo e execute o comando `app_update 4129620 validate`. Isso comparará seus arquivos locais com a versão mais recente nos servidores Steam e baixará apenas os arquivos alterados.

**P: Posso hospedar mais de 8 jogadores?**

R: Embora você possa editar o JSON para permitir mais jogadores, a versão atual de 2026 do Windrose é otimizada para cooperativo em pequenos grupos. Aumentar a contagem além de 12 a 16 jogadores pode levar a dessincronização significativa durante o combate entre navios, a menos que você esteja usando hardware de nível empresarial.
