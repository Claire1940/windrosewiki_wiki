## Понимание SteamDB и App ID Windrose

Прежде чем приступить к установке, необходимо определить правильный идентификатор приложения. SteamDB (Steam Database) — это независимый инструмент, который отслеживает каждое программное обеспечение на платформе Steam. Для выделенных серверов App ID часто отличается от ID основной игры.

Согласно последним данным **steamdb windrose** на 2026 год, компонент выделенного сервера позволяет выполнять анонимный вход, что означает, что вам не обязательно владеть игрой на аккаунте, с которого запускается серверное ПО. Это огромное преимущество для тех, кто использует удаленный VPS-хостинг или дополнительные машины.

| Компонент | Идентификатор |
| :--- | :--- |
| **Windrose Game App ID** | 2431620 (Пример) |
| **Windrose Dedicated Server App ID** | **4129620** |
| **Требование для входа** | Анонимно (Anonymous) |
| **Поддержка платформ** | Windows / Linux (через Proton/Wine) |

## Аппаратные и системные требования

Хостинг выделенного сервера требует больше ресурсов, чем просто игра. Поскольку сервер должен обрабатывать физику мира, поведение ИИ вражеских кораблей и синхронизацию данных игроков, соблюдение минимального порога аппаратного обеспечения жизненно важно для плавной работы.

| Ресурс | Минимальные требования | Рекомендуется (8+ игроков) |
| :--- | :--- | :--- |
| **Процессор** | Четырехъядерный 3.0 ГГц+ | Шестиядерный 3.6 ГГц+ |
| **Память (RAM)** | 8 ГБ | 16 ГБ |
| **Хранилище** | 10 ГБ на SSD | 20 ГБ на NVMe SSD |
| **Скорость отдачи** | 5 Мбит/с | 20 Мбит/с+ |

> **⚠️ Внимание:** Не пытайтесь хостить сервер и играть на одной и той же машине, если у вас менее 32 ГБ оперативной памяти и нет высокопроизводительного многопоточного процессора. Это может привести к значительным задержкам («rubber-banding») и вылетам у всех подключенных игроков.

## Шаг 1: Установка SteamCMD

SteamCMD — это консольная версия клиента Steam. Это стандартный инструмент для загрузки и обновления файлов выделенных серверов. Выполните следующие шаги для подготовки среды:

1.  **Скачайте SteamCMD:** Посетите официальный сайт Valve Developer Community и скачайте версию SteamCMD для Windows.
2.  **Распаковка:** Создайте папку с именем `SteamCMD` в корне вашего диска (например, `C:\SteamCMD`) и извлеките туда содержимое zip-архива.
3.  **Инициализация:** Запустите `steamcmd.exe`. Программа загрузит необходимые обновления и перейдет к командной строке `Steam>`.

## Шаг 2: Загрузка файлов сервера Windrose

После инициализации SteamCMD вам нужно получить файлы, связанные с серверным ID из **steamdb windrose**. Используйте следующую последовательность команд в терминале:

1.  Введите `login anonymous` и нажмите Enter.
2.  Установите директорию установки, введя `force_install_dir ./windrose_server/`.
3.  Запустите загрузку командой `app_update 4129620 validate`.

Флаг «validate» важен, так как он проверяет файлы на наличие ошибок в процессе загрузки. Как только в терминале появится сообщение «Success! App '4129620' fully installed», вы можете ввести `exit`, чтобы закрыть окно.

## Шаг 3: Первый запуск сервера

Перейдите в папку `windrose_server`. Там вы найдете исполняемый файл `WindroseServer.exe` и пакетный файл, обычно называемый `StartServerForeground.bat`. 

**Выполните следующие действия для первого запуска:**
1.  Щелкните правой кнопкой мыши по `StartServerForeground.bat` и выберите **Изменить**.
2.  Убедитесь, что пути указаны верно, и запустите файл.
3.  Дождитесь, пока консоль сгенерирует файлы мира. Вы увидите логи «Collector Loading» и «Event».
4.  Как только мир будет сгенерирован, нажмите `Ctrl + C` в окне консоли, чтобы безопасно завершить работу. Этот шаг необходим для создания конфигурационных JSON-файлов.

## Шаг 4: Настройка параметров сервера

После первого выключения перейдите в директорию `Saved/Config` внутри папки вашего сервера. Здесь вы найдете файл `ServerDescription.json`. В этом файле определяется идентичность вашего сервера.

| Параметр | Описание | Рекомендуемая настройка |
| :--- | :--- | :--- |
| **ServerName** | Имя, отображаемое в браузере серверов. | Название вашего сообщества |
| **MaxPlayerCount** | Общее количество слотов. | **от 8 до 16** |
| **Password** | Оставьте пустым для публичного доступа. | **Надежный буквенно-цифровой** |
| **IsPasswordProtected** | Включает/выключает проверку пароля. | true / false |
| **InviteCode** | Уникальный код для прямого подключения. | **Генерируется автоматически** |

> **💡 Совет:** Всегда сохраняйте резервную копию вашего `InviteCode`. В Windrose этот код часто является самым надежным способом для друзей найти ваш сервер, если глобальный браузер перегружен.

## Раздел 5: Конфигурация сети и проброс портов

Чтобы игроки извне вашей локальной сети могли подключиться, вы должны открыть определенные порты на роутере и в брандмауэре Windows. На этом этапе у большинства хостов возникают сложности.

1.  **Брандмауэр Windows:** Создайте новое правило для входящих подключений для протоколов TCP и UDP.
2.  **Настройки роутера:** Войдите в панель управления роутером и перейдите в раздел Port Forwarding (Проброс портов).
3.  **Порты для открытия:** По умолчанию Windrose обычно использует порты в диапазоне **7777-7778** и порт **27015** для запросов Steam.

Обратитесь к документации производителя вашего роутера, чтобы убедиться, что внутренний IP-адрес вашей серверной машины правильно сопоставлен с этими портами.

Как отмечается в обсуждениях расширенной настройки **steamdb windrose**, можно создать второй, более детальный файл конфигурации для управления игровой механикой. Этот файл позволяет настраивать:

## Обслуживание вашего сервера Windrose

Поддержание сервера в актуальном состоянии жизненно важно для совместимости. Всякий раз, когда разработчики выпускают патч, ваш сервер, скорее всего, станет «несовместимым», пока вы не обновите файлы.

**Процесс обновления:**
1.  Закройте консоль сервера.
2.  Откройте SteamCMD.
3.  Введите `login anonymous`.
4.  Введите `app_update 4129620`.
5.  По завершении перезапустите пакетный файл сервера.

Вы можете автоматизировать это, создав файл «Update.bat», содержащий эти команды, что позволит выполнять обслуживание одним щелчком мыши.

Для получения дополнительной информации об обновлениях игры и статистике игроков посетите [официальную страницу Windrose в Steam](https://store.steampowered.com/app/2431620/Windrose/), где публикуются последние логи разработчиков.

## FAQ (Часто задаваемые вопросы)

**В: Почему мои друзья не видят мой сервер в списке?**

О: Обычно это связано с проблемами проброса портов. Убедитесь, что порты 7777 и 27015 открыты для TCP и UDP. Также проверьте, не блокирует ли брандмауэр Windows файл `WindroseServer.exe`.

**В: Нужно ли мне покупать вторую копию игры для хостинга?**

О: Нет. Используя App ID **steamdb windrose** (4129620) и заходя под «anonymous» в SteamCMD, вы можете захостить сервер без покупки дополнительной лицензии.

**В: Как мне сменить карту мира?**

О: Генерация мира происходит при первом запуске. Если вы хотите начать в новом мире, вам нужно удалить папку `Saved` в директории вашего сервера. При следующем запуске будет сгенерирован новый случайный сид (seed).

**В: Можно ли запустить это на Linux-сервере?**

О: Хотя нативные файлы предназначены для Windows, многие успешно запускают выделенный сервер Windrose на дистрибутивах Linux, используя Proton или Wine. Однако такая настройка значительно сложнее и может потребовать дополнительных зависимостей, таких как `lib32gcc`.
