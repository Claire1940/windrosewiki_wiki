## Зачем использовать Steam DB Windrose для настройки сервера?

Steam DB (Steam Database) — это бесценный сторонний инструмент, предоставляющий подробную информацию о каждом приложении на платформе Steam. При хостинге выделенного сервера для Windrose вам специально потребуется **App ID** для серверного программного обеспечения, который часто отличается от ID игрового клиента.

Ссылаясь на техническую страницу **steam db windrose**, мы можем определить, что выделенный сервер Windrose использует определенный ID для загрузки через SteamCMD. Это гарантирует, что вы загружаете правильные бинарные файлы, оптимизированные для производительности сервера, а не полный игровой клиент, что значительно экономит место на диске и оперативную память.

### Техническая идентификация

| Название ресурса | Steam App ID | Назначение |
|---|---|---|
| **Windrose (Игровой клиент)** | [Варьируется] | Игра в саму игру |
| **Windrose Dedicated Server** | **4129620** | Хостинг постоянного мира |
| **SteamCMD** | Н/А | Консольная утилита для загрузки |

## Минимальные требования к серверу на 2026 год

Прежде чем приступать к загрузке, убедитесь, что ваше оборудование справится с нагрузками мира выживания в открытом море. Windrose требует значительных вычислительных мощностей для расчета физики кораблей и поиска пути ИИ для морских существ.

| Компонент | Рекомендуемые характеристики | Примечания |

| **ЦП** | 4 ядра (3.5GHz+) | Высокая частота на одно ядро предпочтительнее |
| **ОЗУ** | 8GB - 16GB | Сильно зависит от количества игроков |

> **⚠️ Предупреждение:** Не рекомендуется запускать выделенный сервер на той же машине, на которой вы играете. Такая настройка («Listen Server») может привести к значительным задержкам (лагам) и сбоям физики во время интенсивных морских сражений.

## Пошаговая инструкция: Установка через SteamCMD

Чтобы хостить сервер, не владея второй копией игры, вы должны использовать SteamCMD. Эта консольная версия Steam позволяет выполнять анонимный вход для получения файлов сервера.

### 1. Инициализация SteamCMD
Загрузите исполняемый файл SteamCMD с официального сайта Valve Developer Community. Распакуйте папку в отдельный каталог на вашем диске (например, `C:\SteamCMD`). Запустите `steamcmd.exe`, чтобы программа обновилась и создала внутреннюю структуру файлов.

### 2. Загрузка файлов Windrose
Как только вы увидите приглашение `Steam>`, выполните следующую последовательность команд для загрузки файлов сервера, используя ID, найденный на **steam db windrose**:

1.  Введите `login anonymous` и нажмите Enter.
2.  Введите `app_update 4129620 validate` и нажмите Enter.
3.  Дождитесь сообщения "Success! App '4129620' fully installed".
4.  Введите `exit`, чтобы закрыть консоль.

## Настройка файлов сервера

После завершения загрузки перейдите в каталог SteamCMD: `\steamapps\common\Windrose Dedicated Server\`. Эта папка содержит исполняемый файл и пакетные файлы (batch files), необходимые для запуска мира.

### Основная структура файлов
*   **Start server foreground.bat**: Основной скрипт запуска.
*   **Saved/Config/WindowsServer/**: Директория, где хранятся ваши пользовательские настройки.
*   **server description.json**: Файл, используемый для идентификации вашего сервера.

### Редактирование описания сервера
Откройте файл `server description.json` с помощью текстового редактора, например Notepad++. Здесь вы определяете, как ваш сервер будет отображаться в браузере.

| Параметр | Пример значения | Описание |
|---|---|---|
| **ServerName** | "Tortuga Bay 2026" | Имя, отображаемое в списке серверов |
| **MaxPlayerCount** | 8 | Максимальное количество игроков |
| **Password** | "Pirate123" | Оставьте пустым для публичного сервера |
| **IsPasswordProtected** | true / false | Включает/выключает запрос пароля |
| **InviteCode** | [Генерируется автоматически] | Используется для прямого подключения |

> **💡 Совет:** Всегда запускайте сервер один раз, а затем выключайте его с помощью `Ctrl+C` перед редактированием конфигураций. Это гарантирует, что игра создаст необходимую структуру папок и стандартные JSON-файлы для редактирования.

## Сеть и проброс портов

Чтобы друзья могли присоединиться к вашему серверу извне вашей локальной сети, вы должны открыть определенные порты на роутере и в брандмауэре Windows. Если этого не сделать, сервер будет отображаться как «Оффлайн» или будет возникать ошибка «Timed Out» у внешних пользователей.

1.  **Доступ к роутеру**: Введите IP-адрес вашего шлюза (обычно 192.168.1.1) в браузере.
2.  **Найдите Port Forwarding**: Обычно это находится в разделе «Advanced Settings» или «Gaming».
3.  **Проброс портов**: Вам нужно пробросить порты для запросов и связи игры (проверьте файл `README` в папке сервера для уточнения диапазона портов версии 2026 года).
4.  **Правила брандмауэра**: Создайте «Входящее правило» (Inbound Rule) в брандмауэре Windows Defender, чтобы разрешить `WindroseServer.exe` обмениваться данными через вашу сеть.

> **⚠️ Примечание по безопасности:** Открытие портов сопряжено с определенными рисками. Никогда не делитесь прямым IP-адресом вашего сервера на публичных форумах; вместо этого используйте систему кодов приглашения, предусмотренную в интерфейсе Windrose.

## Подключение к выделенному серверу

Как только в окне консоли появится сообщение о том, что мир успешно загружен, игроки могут присоединиться через игровое меню.

1.  Запустите **Windrose**.
2.  Выберите **Play** в главном меню.
3.  Выберите **Connect to a Server**.
4.  Вставьте **Invite Code**, найденный в вашем файле `server description.json`.
5.  Нажмите **Connect**.

Если сервер не отображается, убедитесь, что ваш IP-адрес не изменился и окно `Start server foreground.bat` все еще активно.

## FAQ (Часто задаваемые вопросы)

**В: Где я могу найти актуальный App ID для сервера Windrose?**

О: Вам следует проверить страницу **steam db windrose** в разделе «Apps» или «Depots». По состоянию на 2026 год ID выделенного сервера — 4129620, но крупные обновления игры иногда могут вводить новые экспериментальные ветки.

**В: Нужно ли мне покупать вторую копию Windrose для хостинга сервера?**

О: Нет. Используя SteamCMD и входя как «anonymous», вы можете бесплатно скачать и хостить файлы выделенного сервера. Однако вам все равно нужна купленная копия игры на вашем основном аккаунте, чтобы играть на этом сервере.

**В: Консоль моего сервера застряла на "Collector Loading". Он завис?**

О: Не обязательно. В зависимости от скорости вашего процессора и размера мира, генерация начального макета островов может занять несколько минут. Проверьте диспетчер задач; если использование ЦП процессом активно, сервер все еще работает.

**В: Как обновить сервер Windrose после выхода нового патча?**

О: Снова запустите SteamCMD, войдите анонимно и выполните команду `app_update 4129620 validate`. Это сравнит ваши локальные файлы с последней версией на серверах Steam и загрузит только измененные файлы.

**В: Могу ли я хостить более 8 игроков?**

О: Хотя вы можете отредактировать JSON, чтобы разрешить больше игроков, текущая сборка Windrose 2026 года оптимизирована для небольших групп. Увеличение количества игроков сверх 12-16 может привести к значительной рассинхронизации (desync) во время морских боев, если только вы не используете серверное оборудование корпоративного класса.
